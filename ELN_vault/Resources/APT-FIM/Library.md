# APT/FIM Resources

## PDF intake workflow

1. Drop new PDFs into `Resources/APT-FIM/PDFs` (place them in that folder directly, not inside `Processed` or `Failed`).
2. Start the [[Lab Log Writer|Lab Log Writer]] and open the [Resources panel](http://127.0.0.1:8765/?form=resource-library).
3. Click **Scan Intake Folder** to refresh the queue of PDFs waiting for processing.
4. Before you click **Process Intake Folder**, decide whether to enable **Generate embeddings during ingest**. If you enable it, also choose an `Embedding Model`; if you leave it off, intake still completes without embeddings.
5. Click **Process Intake Folder** to ingest, summarize, and run the same completion steps as manual processing, using the provider and model settings selected in the panel.
6. When a PDF finishes successfully, it is moved to `Resources/APT-FIM/PDFs/Processed`. A PDF only counts as fully finished with embeddings when **Generate embeddings during ingest** was enabled for that run; otherwise it is finished once the non-embedding intake steps complete. If a required step fails, the PDF is moved to `Resources/APT-FIM/PDFs/Failed`.

Topic synthesis and per-PDF controls in the same panel still apply when you want to work on one file at a time.

## Notes

- Intake PDFs start in `Resources/APT-FIM/PDFs`; after batch or manual processing, completed files are under `Resources/APT-FIM/PDFs/Processed` and failures under `Resources/APT-FIM/PDFs/Failed`
- Extracted chunk/index data lives in `Resources/APT-FIM/.index`
- Summary notes are regenerated by the writer when you re-run summarization
- Provider, base URL, model, and embedding-model defaults from the Resources panel persist in the writer's local settings file; API keys persist only in the current browser's local storage
